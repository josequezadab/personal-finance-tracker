# Personal Finance Tracker

A REST API for tracking personal income and expenses, built with FastAPI and SQLite.

## Tech Stack

- **FastAPI** — async REST framework
- **SQLAlchemy 2** — ORM with SQLite
- **Pydantic v2** — request/response validation
- **Uvicorn** — ASGI server
- **Python 3.14**

## Getting Started

All commands are run from the `backend/` directory.

### 1. Set up the environment

```bash
cd backend

# Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

### 2. Run the server

```bash
# from the backend/ directory, with the venv activated
uvicorn app.main:app --reload
```

> The app package is imported as `app` (`from app import models`), so the
> server must be started from inside `backend/`, using the `app.main:app` target.

The API will be available at `http://localhost:8000`.
Interactive docs (Swagger UI) at `http://localhost:8000/docs`.

A `finance.db` SQLite file is created automatically in `backend/` on first run.

## API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/` | Welcome message |
| `GET` | `/transactions` | List all transactions |
| `POST` | `/transactions` | Create a transaction |
| `PUT` | `/transactions/{transaction_id}` | Update a transaction |
| `DELETE` | `/transactions/{transaction_id}` | Delete a transaction |

### Example: Create a transaction

```bash
curl -X POST http://localhost:8000/transactions \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Grocery run",
    "amount": 85.50,
    "category": "Food",
    "type": "expense"
  }'
```

### Example response

```json
{
  "id": 1,
  "description": "Grocery run",
  "amount": 85.50,
  "date": "2026-06-16T10:30:00",
  "category": "Food",
  "type": "expense"
}
```

## Data Model

**Transaction**

| Field | Type | Description |
|-------|------|-------------|
| `id` | int | Auto-generated primary key |
| `description` | string | Short description of the transaction |
| `amount` | float | Transaction amount |
| `date` | datetime | Auto-set to creation time |
| `category` | string | e.g. Food, Transport, Salary |
| `type` | string | `income` or `expense` |

**Request body** (`POST` / `PUT`) requires `description`, `amount`, `category`, and `type`.
The `id` and `date` fields are generated by the server.
